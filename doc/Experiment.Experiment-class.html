The Experiment class is the central component of the botnet emulators architecture. It is responsible for instantiating the topology and the overlord and defining the main steps of the experiment, as well as starting and stopping it. To implement an actual botnet experiment, the class defines some abstract methods that need to be overriden as per the Strategy pattern.

Some of the abstract methods have default implementations that should be called using the super() operator.

Please note that none of the methods in this class are thread-safe.
